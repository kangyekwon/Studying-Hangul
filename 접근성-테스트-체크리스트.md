# 웹 접근성 테스트 체크리스트

## 1. 인식의 용이성 (Perceivable)

### 1.1 대체 텍스트 (Text Alternatives)

- [x] **1.1.1 비텍스트 콘텐츠** (Level A)
  - [x] 모든 이미지와 아이콘에 alt 텍스트 또는 aria-label 제공
  - [x] 장식용 요소에 `aria-hidden="true"` 적용 (bg-particles, confetti)
  - [x] 마스코트 SVG에 aria-label 제공
  - [x] 사운드 버튼에 aria-label 제공

### 1.2 시간 기반 미디어 (Time-based Media)

- [x] **1.2.1 오디오 전용 및 비디오 전용** (Level A)
  - [x] TTS(Text-to-Speech) 음성 출력에 대한 텍스트 표시
  - [x] 음성 듣기 버튼과 함께 텍스트 표시

### 1.3 적응 가능 (Adaptable)

- [x] **1.3.1 정보와 관계** (Level A)
  - [x] 시맨틱 HTML 사용 (`<header>`, `<main>`, `<nav>`)
  - [x] 적절한 ARIA role 사용 (dialog, button, navigation 등)
  - [x] 폼 요소에 레이블 연결

- [x] **1.3.2 의미있는 순서** (Level A)
  - [x] 논리적인 DOM 순서 (헤더 → 통계 → 네비게이션 → 게임 영역)
  - [x] Tab 키 순서가 시각적 순서와 일치

- [x] **1.3.3 감각적 특성** (Level A)
  - [x] 색상에만 의존하지 않는 피드백 제공
  - [x] 정답/오답 시 시각적 + 청각적 + 텍스트 피드백

### 1.4 구별 가능 (Distinguishable)

- [x] **1.4.1 색상 사용** (Level A)
  - [x] 색상 외에도 텍스트, 패턴으로 정보 전달

- [x] **1.4.3 명도 대비 (최소)** (Level AA)
  - [x] 텍스트와 배경 명도 대비 최소 4.5:1
  - [x] 큰 텍스트(18pt 이상)는 최소 3:1
  - [x] 주요 텍스트: 흰색(#ffffff) on 다크 배경(#0a0a1a) - 대비 비율 약 19:1 ✅

- [x] **1.4.11 비텍스트 명도 대비** (Level AA)
  - [x] UI 컴포넌트 테두리와 배경 대비 최소 3:1
  - [x] 버튼 포커스 아웃라인 대비 충분

- [ ] **1.4.12 텍스트 간격** (Level AA)
  - [ ] 사용자가 텍스트 간격 조정 시 레이아웃 유지 확인 필요

## 2. 운용의 용이성 (Operable)

### 2.1 키보드 접근성 (Keyboard Accessible)

- [x] **2.1.1 키보드** (Level A)
  - [x] 모든 기능을 키보드로 조작 가능
  - [x] Enter/Space로 버튼 클릭, 카드 뒤집기
  - [x] Arrow keys로 옵션 이동
  - [x] Tab으로 포커스 이동

- [x] **2.1.2 키보드 트랩 없음** (Level A)
  - [x] 팝업/모달에서 Escape로 빠져나올 수 있음
  - [x] Tab 순환이 자연스럽게 이동

### 2.2 충분한 시간 (Enough Time)

- [ ] **2.2.1 시간 제한 조절** (Level A)
  - [ ] Speed Challenge 타이머 일시정지/조절 기능 필요 (현재 미구현)
  - [ ] Boss Battle 타이머 조절 옵션 추가 권장

### 2.3 발작 및 신체적 반응 (Seizures and Physical Reactions)

- [x] **2.3.1 깜빡임 (3회 이하)** (Level A)
  - [x] 애니메이션이 초당 3회 이하로 깜빡임
  - [x] `prefers-reduced-motion` 지원 (accessibility.css)

### 2.4 탐색 가능 (Navigable)

- [x] **2.4.1 블록 건너뛰기** (Level A)
  - [x] "Skip to main content" 링크 제공

- [x] **2.4.2 페이지 제목** (Level A)
  - [x] `<title>` 태그 제공: "K-POP Korean Learning Game"

- [x] **2.4.3 포커스 순서** (Level A)
  - [x] 논리적인 Tab 순서

- [x] **2.4.4 링크 목적 (문맥상)** (Level A)
  - [x] 모든 버튼에 명확한 레이블

- [x] **2.4.7 포커스 표시** (Level AA)
  - [x] 명확한 포커스 아웃라인 (3px solid, 명도 대비 충분)
  - [x] 포커스 시 박스 그림자 추가로 명확성 향상

### 2.5 입력 방식 (Input Modalities)

- [x] **2.5.1 포인터 제스처** (Level A)
  - [x] 단일 포인터 동작만 사용 (드래그, 멀티터치 없음)

- [x] **2.5.2 포인터 취소** (Level A)
  - [x] 클릭 이벤트가 마우스 다운이 아닌 클릭에 발생

- [x] **2.5.3 레이블 내 이름** (Level A)
  - [x] 버튼 텍스트와 aria-label이 일치하거나 포함

- [x] **2.5.5 타겟 크기** (Level AAA, 권장사항)
  - [x] 모든 터치 타겟 최소 44x44px (accessibility.css에서 보장)

## 자동화 테스트 도구

### 추천 도구

1. **axe DevTools** (Chrome/Firefox 확장)
   - 설치: [https://www.deque.com/axe/devtools/](https://www.deque.com/axe/devtools/)
   - 사용: F12 → axe DevTools 탭

2. **WAVE** (웹 접근성 평가 도구)
   - 사이트: [https://wave.webaim.org/](https://wave.webaim.org/)

3. **Lighthouse** (Chrome DevTools 내장)
   - F12 → Lighthouse 탭 → Accessibility 카테고리

4. **Pa11y**
   - CLI 도구: `npm install -g pa11y`
   - 실행: `pa11y http://localhost:8000`

### 자동화 테스트 명령어

```bash
# Lighthouse 접근성 점수
lighthouse http://localhost:8000 --only-categories=accessibility --output=html --output-path=./lighthouse-report.html

# Pa11y 테스트
pa11y http://localhost:8000 --standard WCAG2AA --reporter cli
```

## 수동 테스트 체크리스트

### 키보드 네비게이션

- [ ] Tab 키로 모든 인터랙티브 요소 접근 가능
- [ ] 포커스가 명확하게 표시됨
- [ ] 논리적인 순서로 포커스 이동
- [ ] Shift+Tab으로 역방향 이동 가능
- [ ] Enter/Space로 모든 버튼 활성화
- [ ] Arrow keys로 옵션 이동
- [ ] Escape로 팝업 닫기
- [ ] 키보드 트랩 없음

### 스크린 리더

- [ ] 모든 텍스트가 읽힘
- [ ] 버튼과 링크가 명확하게 식별됨
- [ ] 이미지 alt 텍스트 읽기
- [ ] ARIA live 영역에서 상태 변경 알림
- [ ] 폼 레이블이 올바르게 연결됨

### 시각적 접근성

- [ ] 명도 대비 충분 (텍스트, 버튼, 아이콘)
- [ ] 색맹 모드에서 정보 구분 가능
- [ ] 200% 확대 시 레이아웃 유지
- [ ] 작은 화면(모바일)에서 터치 타겟 충분

### 모션 감소

- [ ] `prefers-reduced-motion` 설정 시 애니메이션 감소
- [ ] 중요한 정보는 모션 없이도 전달
